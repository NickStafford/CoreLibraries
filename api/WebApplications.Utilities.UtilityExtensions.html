    The second enumerable object
    IEqualityComparer<T> comparer

    The comparer.
    Returns
    Type Description
    Boolean

    Returns true if both of the enumerable objects are the same size and contain the same elements.
    Type Parameters
    Name Description
    T

    The type of the object contained in the enumerable.
    Remarks

    This does a robust check of enumerable equality with an algorithm that's no worse that O(N).

    The order in which items appear in the enumerable doesn't matter. For example {1, 2, 2, 3} would be considered equal to {2, 1, 3, 2} but not equal to {1, 2, 3, 3}.

    If the list does not contain duplicate items use DeepEqualsSimple.

    If the lists are sorted then use Enumerable's Sequence Equal.
