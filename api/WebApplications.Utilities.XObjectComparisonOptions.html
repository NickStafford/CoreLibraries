    Comparison options for .

    [Flags]

public enum XObjectComparisonOptions

    Fields

    Name Description
    Default

    The default options.
    Exact

    Everything is considered significant.
    IgnoreAdditional

    The order of elements is ignored, and additional nodes are ignored.
    IgnoreAdditionalAttributes

    Attributes that occur in the second node but don't occur in the first node are ignored (Also sets IgnoreAttributeOrder).
    IgnoreAdditionalComments

    Comments that occur in the second node but don't occur in the first node are ignored (Also sets IgnoreCommentOrder).
    IgnoreAdditionalDocumentTypes

    Document types that occur in the second node but don't occur in the first node are ignored .
    IgnoreAdditionalElements

    Child elements that occur in the second node but don't occur in the first node are ignored (Also sets IgnoreElementOrder).
    IgnoreAdditionalProcessingInstructions

    Processing instructions that occur in the second node but don't occur in the first node are ignored (Also sets IgnoreProcessingInstructionOrder).
    IgnoreAdditionalText

    Text/CDATA that occurs in the second node but doesn't occur in the first node are ignored (Also sets IgnoreTextOrder).
    IgnoreAttributeOrder

    The attribute order is not considered significant.
    IgnoreAttributes

    All attributes are ignored.
    IgnoreCommentOrder

    The comment order is not considered significant.
    IgnoreComments

    All comments are ignored
    IgnoreDocumentTypes

    All document types are ignored.
    IgnoreElementOrder

    The child element order is not considered significant.
    IgnoreElements

    All child elements are not considered.
    IgnoreProcessingInstructionOrder

    The order of processing instructions is ignored.
    IgnoreProcessingInstructions

    All processing instructions are ignored.
    IgnoreText

    All text/CDATA is ignored
    IgnoreTextOrder

    The text/CDATA order is not considered significant.
    IgnoreTextOutsideOfChildren

    All text/CDATA that occurs outside of child elements is ignored, but text in nodes without child elements is considered.
    Legacy

    The legacy option is most similar to DeepEquals(XNode, XNode).
    Loose

    Order does not matter, text outside of child nodes, comments, processing instructions and document types are all ignored.
    None

    Everything is considered significant.
    Normalised

    The order of elements is ignored.
    Semantic

    The semantic mode ensures that the second node is effectively compatible with the first node, i.e. it contains all the nodes in the first, but not necessarily in the same order, additonal nodes are allowed, and insignificant nodes are ignored.
