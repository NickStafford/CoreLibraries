    PreservesLength

    true if the cryptography provider preserves length during encryption/decryption.

    Declaration
    [PublicAPI]
public readonly bool PreservesLength
    Field Value
    Type Description
    Boolean
    Remarks

    When using the GetEncryptionStream(Stream), GetDecryptionStream(Stream), GetEncryptor() and/or GetDecryptor() methods to perform encryption/decryption, care should be taken to observe the value of this flag, as the encrypted/decrypted stream may not be the same length as the original input stream, and may have trailing zero bytes.

    All other encryption/decryption methods respect this flag and will automatically mark the end of an input stream.

    GetDecryptionStream(Stream)

    Gets the decryption stream.

    Declaration
    [PublicAPI]
public CryptoStream GetDecryptionStream(Stream inputStream)
    Parameters
    Type Name Description
    Stream inputStream

    The input stream.
    Returns
    Type Description
    CryptoStream

    A CryptoStream that can be used to read decrypted data from the inputStream.
    Remarks

    To decrypt data, write it to the inputStream and read it from the CryptoStream.

    When using a provider that does not PreservesLength you may want to consider adding a termination byte (e.g. 0xFF) to help detect the end of the stream.

    Exceptions
    Type Condition
    ArgumentNullException

    inputStream is null.
    CryptographicException

    The cryptographic provider cannot perform decryption.
