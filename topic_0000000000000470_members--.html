CachingDictionary
CachingDictionary<TKey, TValue> A thread safe dictionary that caches and keeps track of elements in the cache. This allows for greater functionality than the EnhancedMemoryCache<TKey, TValue>, including, key searches, non-string keys, etc.
CachingDictionaryBase<TKey, TValue> Implements core functionality for caching classes.
CachingQueue<T> Implements a thread safe FIFO queue of objects that will only remain present for a specific duration.
ConcurrentLookup<TKey, TValue> Implements a concurrent lookup, which allows a set of objects to be grouped and manipulated by a key.
CyclicConcurrentQueue<T> Represents a thread-safe first in-first out (FIFO) collection. As the size exceeds Capacity the queue automatically throws away head items.
DictionaryEnumerator<TKey, TValue> A dictionary enumerator wraps the normal enumerator.
EnhancedMemoryCache
EnhancedMemoryCache<TKey, TValue> An Enhanced Memory Cache implementation, uses the default Memory Cache, but allows multiple instances to be created (they all map share the underlying memory cache). This ensures type-safety on the cache values but does not allow the value to be a null reference (Nothing in Visual Basic). For null value support you can use EnhancedMemoryCacheNull<TKey,TValue>, though there is a small performance and memory overhead in doing so.
EnhancedMemoryCache<TValue> Obsolete. OBSOLETE - Use EnhancedMemoryCache<TKey,Tvalue>
EnhancedMemoryCacheNull<TKey, TValue> An Enhanced Memory Cache implementation, uses the default Memory Cache but allows multiple instances to be created (they all map share the underlying memory cache). This ensures type-safety on the cache values and allows the stored value to be a null reference (Nothing in Visual Basic). There is a small performance penalty for this support so the EnhancedMemoryCache<TKey,TValue> should be used in preference to this class in the case where nulls (null reference (Nothing in Visual Basic)) are not possible.
EnhancedMemoryCacheNull<TValue> Obsolete. OBSOLETE - Use EnhancedMemoryCacheNull<TKey,Tvalue>
ObservableWeakReference<T> A potentially observable weak reference that allows notification when a reference is finalized, as long as we are not tracking for resurrection and the type implements IObservableFinalize.
WeakConcurrentDictionary<TKey, TValue> Synchronized dictionary keeping only weak references to the items (but not to their keys) it contains. If TValue implements IObservableFinalize then items are removed automatically, otherwise they are only removed when attempting to access (e.g. through iterating through the collection, or 'counting', etc.)
WeakConcurrentLookup<TKey, TValue> A concurrent lookup, which allows a set of weak referenced objects to be grouped and manipulated by a key. If TValue implements IObservableFinalize then items are removed automatically, otherwise they are only removed when attempting to access (e.g. through iterating through the collection, or 'counting', etc.)
WeakReference<T> A type safe weak reference.
WeakSingleton<T> Holds a WeakReference<T> to an instance of an object, lazily initialising the value if the object has been garbage collected.
